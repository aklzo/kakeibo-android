# 画面仕様

## BottomNavigation 構成

| インデックス | アイコン | ラベル | 画面 |
|---|---|---|---|
| 0 | Add | 追加 | 取引追加 |
| 1 | List | 一覧 | 取引一覧 |
| 2 | BarChart | 集計 | 月次集計 |
| 3 | TrendingUp | 進捗 | 消費進捗率 |
| 4 | AccountBalance | 予算 | 予算設定 |

---

## 1. 取引追加画面

API: `POST /api/v1/transactions`

### 入力項目

| 項目 | 種別 | 必須 | デフォルト | バリデーション |
|------|------|------|-----------|---------------|
| 名称 | テキスト | ✓ | - | 空文字不可 |
| 金額 | 数値 | ✓ | - | 正の整数 |
| カテゴリ | ドロップダウン | ✓ | 固定費 | - |
| 日付 | DatePicker | ✓ | 実行日 | - |
| メモ | テキスト | - | - | - |

### 動作

- 送信成功: Snackbar で「登録しました」と表示し、フォームをリセットする
- 送信失敗: Snackbar でエラーメッセージを表示する
- 送信中: ボタンを非活性にしてスピナーを表示する

---

## 2. 取引一覧画面

API: `GET /api/v1/transactions`, `PATCH /api/v1/transactions/:id`, `DELETE /api/v1/transactions/:id`

### 表示内容

- 月ナビゲーター（左矢印 / 年月表示 / 右矢印）
- カテゴリフィルター（ドロップダウン、「すべて」含む）
- 取引リスト（日付・名称・金額・カテゴリ表示）

### 編集・削除

- 取引行をタップ → ModalBottomSheet で編集フォームを表示
- 編集フォームの入力項目は追加画面と同じ（既存値で初期化）
- ゴミ箱アイコンタップ → シートを閉じてから削除確認ダイアログを表示
- 削除確認後 DELETE を呼び出し、一覧をリフレッシュする

---

## 3. 月次集計画面

API: `GET /api/v1/summary?by_category=true`

### 表示内容

- 月ナビゲーター
- 収入・支出・収支のサマリーカード（`ElevatedCard`）
  - 収支がプラスの場合は「+」プレフィックスを付ける
- カテゴリ別内訳（カテゴリ名・金額・横棒グラフ）
  - バーの長さは最大カテゴリの金額を基準に正規化する

---

## 4. 消費進捗率画面

API: `GET /api/v1/progress`

### 操作

- 月ナビゲーター（当月より未来には進めない）
- モード切替セグメントボタン（「予算比」/ 「昨月比」）

### 表示内容

- 月全体セクション: 支出 / 基準額・進捗率・LinearProgressIndicator
- カテゴリ別セクション: カテゴリ名・支出 / 基準額・進捗率・LinearProgressIndicator

### 進捗バーの色

| 進捗率 | 色 |
|--------|-----|
| 100% 超 | error（赤） |
| 80% 超〜100% | warning（オレンジ `#F57C00`） |
| 〜80% | primary（青） |

---

## 5. 予算設定画面

API: `GET /api/v1/budgets`, `POST /api/v1/budgets`

### 表示内容

- 月全体セクション: 金額入力フィールド + 設定ボタン
- カテゴリ別セクション: カテゴリごとに金額入力フィールド + 設定ボタン

### 動作

- 画面表示時に GET /budgets を呼び出し、`month == null` のエントリで初期値を設定する
- 設定ボタン押下で対象カテゴリの予算のみ POST する（行ごとに独立して保存）
- 保存成功: Snackbar で「保存しました」と表示する
- バリデーション: 空・ゼロ・負数の場合はエラー Snackbar を表示する
